## Get Started with HiveMQ CE

Follow these steps to get HiveMQ Community Edition running quickly.

<Steps>
  <Step title="Download or Pull Image">
    Choose your preferred method to get HiveMQ CE:
    
    <CodeGroup>
    ```bash Docker
    docker pull hivemq/hivemq-ce
    ```
    
    ```bash Binary Download
    # Download from GitHub Releases
    wget https://github.com/hivemq/hivemq-community-edition/releases/download/2025.5/hivemq-ce-2025.5.zip
    unzip hivemq-ce-2025.5.zip
    cd hivemq-ce-2025.5
    ```
    </CodeGroup>
    
    <Note>
      Java 11 or higher is required to run the binary package.
    </Note>
  </Step>
  
  <Step title="Start HiveMQ CE">
    Launch the broker:
    
    <CodeGroup>
    ```bash Docker
    docker run -d --name hivemq-ce -p 1883:1883 hivemq/hivemq-ce
    ```
    
    ```bash Linux/macOS
    bin/run.sh
    ```
    
    ```bash Windows
    bin\run.bat
    ```
    </CodeGroup>
    
    HiveMQ CE starts on port **1883** (MQTT) and **8000** (WebSocket) by default.
  </Step>
  
  <Step title="Connect a Client">
    Test the connection with an MQTT client:
    
    <Tabs>
      <Tab title="Mosquitto CLI">
        ```bash
        # Subscribe to test topic
        mosquitto_sub -h localhost -p 1883 -t "test/topic"
        
        # Publish a message (in another terminal)
        mosquitto_pub -h localhost -p 1883 -t "test/topic" -m "Hello HiveMQ!"
        ```
      </Tab>
      
      <Tab title="Python">
        ```python
        import paho.mqtt.client as mqtt
        
        def on_connect(client, userdata, flags, rc):
            print("Connected!")
            client.subscribe("test/topic")
        
        def on_message(client, userdata, message):
            print(f"Received: {message.payload.decode()}")
        
        client = mqtt.Client()
        client.on_connect = on_connect
        client.on_message = on_message
        
        client.connect("localhost", 1883, 60)
        client.loop_forever()
        ```
      </Tab>
      
      <Tab title="JavaScript">
        ```javascript
        const mqtt = require('mqtt');
        
        const client = mqtt.connect('mqtt://localhost:1883');
        
        client.on('connect', () => {
          console.log('Connected!');
          client.subscribe('test/topic');
          client.publish('test/topic', 'Hello HiveMQ!');
        });
        
        client.on('message', (topic, message) => {
          console.log(`Received: ${message.toString()}`);
        });
        ```
      </Tab>
    </Tabs>
  </Step>
  
  <Step title="Verify">
    Check that messages are flowing:
    
    - Subscribe to a topic in one terminal
    - Publish to the same topic in another terminal
    - Verify the message is received
    
    <Tip>
      You can also check the HiveMQ CE logs to confirm clients are connecting successfully.
    </Tip>
  </Step>
</Steps>

## What's Running?

Once started, HiveMQ CE provides:

- **MQTT Broker** on port 1883 (TCP)
- **WebSocket Endpoint** on port 8000 at `/mqtt`
- **Local file-based persistence** for QoS 1/2 messages
- **Default configuration** allowing all clients

## Common Issues

<AccordionGroup>
  <Accordion title="Port 1883 already in use">
    Another MQTT broker or service is using port 1883.
    
    **Solution**: Stop the other service or configure HiveMQ CE to use a different port in `config.xml`.
  </Accordion>
  
  <Accordion title="Java version error">
    HiveMQ CE requires Java 11 or higher.
    
    **Solution**: Install JDK 11+ and verify with `java -version`.
  </Accordion>
  
  <Accordion title="Permission denied">
    The run script or binary doesn't have execute permissions.
    
    **Solution**: Run `chmod +x bin/run.sh` on Linux/macOS.
  </Accordion>
</AccordionGroup>
